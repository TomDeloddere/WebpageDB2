Complete Answer Guide :: Mainframe Edition

 Introduction & Principles

1. Which security principle advocates granting users only the minimum access rights necessary for their job?
Answer: Principle of Least Privilege

The Principle of Least Privilege advocates granting users only the minimum access rights necessary to perform their job functions. This reduces the risk of unauthorized access or accidental misuse, enhancing overall security in DB2 and z/OS environments.

2. The primary authentication mechanism for most DB2 for z/OS installations relies on ________.
Answer: RACF

Most DB2 for z/OS installations rely on RACF (Resource Access Control Facility) as the primary authentication mechanism. RACF, or equivalent z/OS security products like ACF2 or TopSecret, validates user credentials and integrates with DB2 to enforce authentication policies.

3. True or False: DB2 security primarily focuses on database-level controls and does not integrate with z/OS security.
Answer: False

DB2 security integrates closely with z/OS security mechanisms, such as RACF, ACF2, or TopSecret. It leverages z/OS for authentication, authorization, and auditing, combining database-level controls with system-level security to provide a robust security framework.

4. Which identifier represents the user's main identity, typically derived from the z/OS security product?
Answer: Primary Authorization ID

The Primary Authorization ID represents the user's main identity in DB2, typically derived from the z/OS security product (e.g., RACF user ID). It is used to determine the user's privileges and access rights within the DB2 subsystem.

 Authorization & Privileges

5. Which DB2 system authority grants complete administrative control over the entire DB2 subsystem?
Answer: SYSADM

The SYSADM (System Administrator) authority grants complete administrative control over the entire DB2 subsystem. It includes the ability to manage all objects, execute utilities, and perform system-level tasks, making it the highest level of authority in DB2.

6. The authority required to grant and revoke privileges on objects is ________.
Answer: ACCESSCTRL

The ACCESSCTRL authority allows users to grant and revoke privileges on DB2 objects, such as tables, views, and stored procedures. This authority is typically assigned to security administrators to manage access rights without requiring full SYSADM privileges.

7. Which object-level privilege allows modifying existing rows in a table?
Answer: UPDATE

The UPDATE privilege allows users to modify existing rows in a table using SQL UPDATE statements. This privilege is specific to the table and can be granted at the table or column level to control data modification access.

8. True or False: The `DBADM` authority provides the same level of control as `SYSADM`, but limited to a specific database.
Answer: True

The DBADM (Database Administrator) authority provides administrative control similar to SYSADM, but it is limited to a specific database. DBADM users can manage objects and privileges within their assigned database without affecting other databases or the subsystem.

9. Granting the `EXECUTE` privilege allows a user to run ________.
Answer: stored procedures

The EXECUTE privilege allows a user to run stored procedures or user-defined functions in DB2. This privilege is necessary for executing these objects and can be granted to specific users, roles, or groups to control access to procedural logic.

10. The `WITH GRANT OPTION` clause allows the recipient of a privilege to ________.
Answer: Grant the same privilege to others

The `WITH GRANT OPTION` clause allows the recipient of a privilege to grant the same privilege to other users or roles. This enables delegated administration, but it must be used cautiously to prevent unintended escalation of access rights.

 Advanced Access Control (LBAC, RCAC)

11. Which access control mechanism uses security labels assigned to data and users to enforce multi-level security?
Answer: Label-Based Access Control (LBAC)

Label-Based Access Control (LBAC) uses security labels assigned to data rows and users to enforce multi-level security. It ensures that users can only access data with compatible security labels, providing fine-grained control for sensitive environments.

12. Defining rules that filter which rows a user can access based on predicates is achieved using ________.
Answer: Row Permissions

Row Permissions, part of DB2's Row and Column Access Control (RCAC), define predicate-based rules to filter which rows a user can access. These rules are enforced transparently, restricting access to specific rows based on user attributes or session context.

13. True or False: Column masks allow hiding entire columns from unauthorized users.
Answer: False

Column masks in DB2 do not hide entire columns; instead, they transform or mask the values within columns for unauthorized users, such as replacing sensitive data with asterisks or nulls. This allows controlled access while protecting sensitive information.

14. Row and Column Access Control (RCAC) features include Row Permissions and ________.
Answer: Column Masks

Row and Column Access Control (RCAC) in DB2 includes Row Permissions, which filter accessible rows, and Column Masks, which transform or obscure column values for unauthorized users. Together, these features provide fine-grained data access control.

15. Creating a security policy with defined levels and compartments is part of implementing ________.
Answer: LBAC

Implementing Label-Based Access Control (LBAC) involves creating a security policy with defined levels and compartments. These policies assign security labels to data and users, enforcing access restrictions based on hierarchical or compartmentalized security requirements.

 Roles & Trusted Contexts

16. A collection of privileges grouped together for a specific job function is called a ________.
Answer: Role

A Role in DB2 is a collection of privileges and authorities grouped together to represent a specific job function. Roles simplify access management by allowing privileges to be assigned to roles, which can then be granted to users or groups.

17. Which DB2 feature allows assigning roles or different security attributes based on connection properties like IP address?
Answer: Trusted Contexts

Trusted Contexts in DB2 allow the assignment of roles or security attributes based on connection properties, such as IP address, user ID, or application name. This enhances security for specific connection scenarios, like application servers or connection pooling.

18. True or False: Roles in DB2 can only be assigned privileges, not system or database authorities.
Answer: False

Roles in DB2 can be assigned both privileges (e.g., SELECT, UPDATE) and system or database authorities (e.g., DBADM, SYSCTRL). This flexibility allows roles to encapsulate a wide range of permissions, simplifying security administration.

19. Trusted contexts can enhance security for connection pooling by associating specific attributes with ________.
Answer: Application server IDs

Trusted contexts enhance security for connection pooling by associating specific attributes, such as roles or privileges, with application server IDs or other connection properties. This ensures that pooled connections operate with appropriate security settings, reducing the risk of privilege misuse.

 User Rights Management & Administration

20. Which administrative model involves security administrators handling all privilege assignments?
Answer: Centralized

The Centralized administrative model involves security administrators handling all privilege assignments, ensuring consistent and controlled access management. This model contrasts with decentralized models where object owners or other users may grant privileges.

21. Regularly reviewing access rights helps prevent privilege ________, where users accumulate unnecessary permissions over time.
Answer: accumulation

Regularly reviewing access rights prevents privilege accumulation (or privilege creep), where users accumulate unnecessary permissions over time due to role changes or project reassignments. This practice ensures adherence to the Principle of Least Privilege and reduces security risks.

22. True or False: The SECADM authority is typically separated from SYSADM to enforce separation of duties.
Answer: True

The SECADM (Security Administrator) authority is typically separated from SYSADM to enforce separation of duties. SECADM focuses on managing security policies and privileges, while SYSADM handles system administration, reducing the risk of a single user having excessive control.

23. Automating user rights management often involves integration with ________ systems for provisioning.
Answer: HR

Automating user rights management often involves integration with HR (Human Resources) systems for provisioning. HR systems provide data on employee roles, hires, and terminations, enabling automated assignment or revocation of DB2 privileges based on job functions.

 Compliance, Auditing & Encryption

24. Which regulation focuses on the protection of personal data for individuals within the European Union?
Answer: GDPR

The General Data Protection Regulation (GDPR) focuses on protecting personal data for individuals within the European Union. It imposes strict requirements on data handling, privacy, and security, affecting DB2 environments that store or process EU residents' data.

25. DB2's capability to record security-relevant events relies on starting specific ________ trace classes.
Answer: Audit

DB2's auditing capability relies on starting specific Audit trace classes to record security-relevant events, such as privilege usage, authentication attempts, or data access. These traces generate IFCID records, which can be analyzed to ensure compliance and detect unauthorized activity.

26. True or False: DB2 native encryption protects data only while it is stored on disk (at rest).
Answer: True

DB2 native encryption protects data at rest, encrypting data stored on disk to prevent unauthorized access. Encryption in transit, such as between clients and the DB2 server, requires separate mechanisms like SSL/TLS, which are not part of native encryption.

27. Protecting data transmitted over the network between clients and the DB2 server often involves using protocols like ________.
Answer: SSL

Protecting data transmitted over the network between clients and the DB2 server often involves using SSL (Secure Sockets Layer) or its successor, TLS (Transport Layer Security). These protocols encrypt network traffic to ensure data confidentiality and integrity.

28. Which DB2 catalog table is commonly queried to review granted object privileges?
Answer: SYSIBM.SYSTABAUTH

The SYSIBM.SYSTABAUTH catalog table is commonly queried to review granted object privileges for tables and views. It records privileges like SELECT, INSERT, or UPDATE, along with the grantee and grantor, aiding in access rights audits.

29. The Payment Card Industry Data Security Standard is commonly referred to as ________.
Answer: PCI DSS

The Payment Card Industry Data Security Standard (PCI DSS) is a set of security standards designed to protect cardholder data. DB2 environments handling payment card data must comply with PCI DSS requirements, including encryption, access controls, and auditing.

 Network Security & Best Practices

30. Securing the Distributed Data Facility (DDF) often involves configuring ________ ports and potentially using AT-TLS.
Answer: Secure (SECPORT)

Securing the Distributed Data Facility (DDF) involves configuring the Secure Port (SECPORT) for encrypted connections and potentially using Application Transparent Transport Layer Security (AT-TLS). This ensures that remote client connections to DB2 are encrypted and secure.

31. Regularly applying IBM-provided maintenance (PTFs) is crucial for addressing known security ________.
Answer: vulnerabilities

Regularly applying IBM-provided Program Temporary Fixes (PTFs) is crucial for addressing known security vulnerabilities in DB2. These updates patch potential exploits, ensuring the subsystem remains secure against evolving threats.

32. True or False: Security documentation should be kept confidential and accessible only to the SYSADM.
Answer: False

Security documentation should be accessible to relevant personnel, such as security administrators, DBAs, and incident response teams, not just SYSADM. While it should be kept confidential from unauthorized users, it must be available for operational and emergency purposes.

33. Which security concept assumes that no user or connection should be implicitly trusted, requiring verification at every step?
Answer: Zero Trust

The Zero Trust security concept assumes that no user or connection should be implicitly trusted, requiring verification at every step. In DB2, this involves continuous authentication, authorization, and monitoring to protect against internal and external threats.

34. Implementing strong password policies and encouraging the use of ________ enhances authentication security.
Answer: password phrases

Implementing strong password policies and encouraging the use of password phrases (longer, complex passphrases) enhances authentication security. Password phrases are harder to crack than traditional passwords, improving protection against unauthorized access in DB2 environments.

35. True or False: DB2 auditing automatically captures all SQL statements executed by default.
Answer: False

DB2 auditing does not automatically capture all SQL statements by default. Specific audit trace classes and policies must be defined and started to record desired events, such as SQL execution, privilege usage, or security violations, ensuring targeted auditing.

36. Which authority is specifically responsible for administering security policies like RCAC and LBAC?
Answer: SECADM

The SECADM (Security Administrator) authority is responsible for administering security policies like Row and Column Access Control (RCAC) and Label-Based Access Control (LBAC). SECADM users manage these advanced access controls to enforce fine-grained security in DB2.
