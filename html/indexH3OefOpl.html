 Requirements & Conceptual/Logical Design

1. Which database design phase focuses on business concepts like entities and relationships, independent of a specific DBMS?
Answer: Conceptual Design

The Conceptual Design phase focuses on defining the high-level structure of the database in terms of entities, their attributes, and relationships, without tying it to a specific DBMS like DB2. This phase produces an Entity-Relationship (ER) model that represents business requirements in a platform-independent manner, serving as the foundation for subsequent logical and physical design phases.

2. The process of eliminating repeating groups and ensuring atomic values corresponds to which Normal Form?
Answer: 1NF

First Normal Form (1NF) requires that all attributes in a table contain atomic (indivisible) values and that repeating groups are eliminated. This ensures each column holds a single value per row, and each row is unique, forming the basis for further normalization to reduce redundancy and improve data integrity.

3. True or False: Denormalization involves adding controlled redundancy to improve query performance.
Answer: True

Denormalization is the intentional introduction of controlled redundancy into a database design to enhance query performance. By reducing the need for complex joins or improving data locality, denormalization can speed up read-heavy operations, though it may complicate updates and increase storage requirements.

4. Which type of integrity constraint ensures that foreign key values match existing primary key values in the referenced table?
Answer: Referential Integrity

Referential Integrity ensures that foreign key values in a child table correspond to existing primary key values in the referenced parent table. This constraint maintains valid relationships between tables, preventing orphaned records and enforcing data consistency in DB2 through foreign key constraints.

5. Eliminating transitive dependencies (where non-key attributes depend on other non-key attributes) is the goal of ________ Normal Form.
Answer: 3NF

Third Normal Form (3NF) eliminates transitive dependencies, ensuring that non-key attributes depend only on the primary key. This reduces data redundancy and prevents update anomalies, building on 2NF by ensuring all attributes are directly dependent on the key, the whole key, and nothing but the key.

 Physical Design for DB2

6. Which DB2 object maps logical tables to physical DASD storage?
Answer: Tablespace

A Tablespace in DB2 is the object that maps logical tables to physical DASD (Direct Access Storage Device) storage. It defines the storage structure for one or more tables, specifying how data is organized and stored on disk, including attributes like page size and partitioning strategy.

7. An index that determines the physical storage order of rows in a table is called a ________ index.
Answer: Clustering

A Clustering index determines the physical storage order of rows in a table, optimizing data retrieval for queries that access data in the order of the index. DB2 attempts to maintain this order during data modifications, improving performance for range-based queries and sequential access.

8. True or False: Range partitioning is typically based on system-generated sequence numbers.
Answer: False

Range partitioning is typically based on meaningful column values, such as dates, codes, or numeric ranges, rather than system-generated sequence numbers. This allows data to be logically grouped into partitions based on business-relevant criteria, improving query performance and manageability.

9. Which DB2 parameter controls the percentage of space left empty on a page during LOAD or REORG for future updates?
Answer: PCTFREE

The PCTFREE parameter in DB2 specifies the percentage of space left empty on each page during LOAD or REORG operations to accommodate future updates or inserts. This free space helps prevent page splits and maintains performance by allowing rows to grow without requiring immediate reorganization.

10. Optimizing a database schema specifically for reporting and analytics often involves using a ________ schema design.
Answer: Star

A Star schema is commonly used for optimizing database schemas for reporting and analytics. It features a central fact table surrounded by dimension tables, simplifying queries and improving performance for analytical workloads. Snowflake schemas, an extension of Star schemas, are also used but are more normalized.

 Database Objects

11. Which DB2 object provides a logical grouping of related objects like tables and views within a database?
Answer: Schema

A Schema in DB2 is a logical grouping of related database objects, such as tables, views, indexes, and stored procedures. It provides a namespace for organizing objects and managing access permissions, allowing multiple users or applications to work within the same database without naming conflicts.

12. A virtual table derived from one or more base tables using a stored SELECT statement is a ________.
Answer: View

A View in DB2 is a virtual table defined by a stored SELECT statement. It does not store data physically but dynamically retrieves data from underlying base tables when queried, providing a way to simplify complex queries, enforce security, or present data in a customized format.

13. True or False: An Alias in DB2 is primarily used to define alternative access paths for performance.
Answer: False

An Alias in DB2 is primarily used to provide an alternative name for a table or view, often for simplifying access, ensuring location transparency, or supporting distributed environments. It does not define alternative access paths for performance, which is typically handled by indexes or query optimization.

14. Which type of temporary table is specific to a single application process and implicitly dropped?
Answer: Declared Temporary Table

A Declared Temporary Table (DTT) in DB2 is specific to a single application process and is implicitly dropped when the session ends. Unlike Global Temporary Tables, DTTs are not shared across sessions and are useful for temporary data storage during complex application processing.

15. Procedural logic automatically executed by DB2 in response to INSERT, UPDATE, or DELETE operations is implemented using ________.
Answer: Triggers

Triggers in DB2 are procedural logic defined to execute automatically in response to INSERT, UPDATE, or DELETE operations on a table. They are used to enforce business rules, maintain data integrity, or automate tasks like logging changes, and can be defined as BEFORE, AFTER, or INSTEAD OF triggers.

 Indexes and Views

16. Which clause in `CREATE INDEX` allows including additional columns in the index leaf pages to satisfy queries without accessing the table?
Answer: INCLUDE

The INCLUDE clause in the `CREATE INDEX` statement allows additional non-key columns to be stored in the index leaf pages. This enables DB2 to satisfy certain queries directly from the index without accessing the underlying table, improving query performance through index-only access.

17. True or False: A table can have multiple clustering indexes defined.
Answer: False

A table in DB2 can have only one clustering index, as it determines the physical storage order of the table's rows. Multiple clustering indexes would conflict with the need for a single physical ordering, though a table can have multiple non-clustering indexes for other purposes.

18. A view created with the `WITH CHECK OPTION` clause restricts ________ operations on the view.
Answer: INSERT

The `WITH CHECK OPTION` clause in a view definition restricts INSERT and UPDATE operations on the view to ensure that modified or inserted rows satisfy the view's WHERE condition. This enforces data integrity by preventing rows that would not be visible through the view from being added or modified.

19. Which DB2 object physically stores pre-computed results of a query, often used to optimize complex views?
Answer: Materialized Query Table (MQT)

A Materialized Query Table (MQT) in DB2 physically stores the pre-computed results of a query, often used to optimize complex views or analytical queries. Unlike regular views, MQTs store data on disk, and their contents can be refreshed manually or automatically to reflect changes in the underlying tables.

 Referential Integrity

20. In Declarative Referential Integrity (DRI), which `ON DELETE` rule automatically deletes child rows when the corresponding parent row is deleted?
Answer: CASCADE

The `ON DELETE CASCADE` rule in Declarative Referential Integrity (DRI) automatically deletes child rows in a dependent table when the corresponding parent row is deleted. This ensures referential integrity by preventing orphaned records and is commonly used when child records are meaningless without their parent.

21. True or False: DB2 directly supports `ON UPDATE CASCADE` using declarative foreign key constraints.
Answer: False

DB2 does not directly support `ON UPDATE CASCADE` using declarative foreign key constraints. To implement cascading updates (where changes to a parent table's primary key propagate to child table foreign keys), triggers or application logic must be used, as declarative constraints only support `ON DELETE` actions like CASCADE, SET NULL, or RESTRICT.

22. The DB2 utility often used to check for and report referential integrity violations is ________.
Answer: CHECK DATA

The CHECK DATA utility in DB2 is used to verify referential integrity and check constraints, reporting any violations. It scans tables to ensure that foreign key values match primary key values in referenced tables and can place tablespaces in a CHECK-pending status if violations are found.

23. Setting a foreign key constraint to `NOT ENFORCED` temporarily disables RI checking but may impact ________.
Answer: Query Optimization

Setting a foreign key constraint to `NOT ENFORCED` disables referential integrity checking, which can be useful during data loads or migrations. However, this may impact query optimization, as the DB2 optimizer relies on enforced constraints to make assumptions about data relationships and choose efficient access paths.

 Advanced Implementation & Constraints

24. Which partitioning strategy automatically adds new partitions as data grows, simplifying capacity management?
Answer: Partition-By-Growth

Partition-By-Growth is a DB2 partitioning strategy that automatically adds new partitions as data grows, simplifying capacity management. Unlike range or list partitioning, it does not require predefined partition boundaries, making it ideal for tables with unpredictable growth patterns.

25. A constraint that ensures values in a column or set of columns are unique but allows NULLs is a ________ constraint.
Answer: Unique

A Unique constraint in DB2 ensures that values in a column or set of columns are unique across all rows in a table, but unlike primary keys, it allows NULL values. This is useful for enforcing uniqueness for optional attributes, such as employee IDs that may not always be assigned.

26. True or False: Check constraints can only validate conditions based on a single column's value.
Answer: False

Check constraints in DB2 can validate complex conditions involving multiple columns within the same row, not just single-column values. For example, a check constraint could ensure that an employee's salary is greater than their department's minimum salary, referencing multiple columns in the condition.

27. DB2's feature for fine-grained security that filters rows based on user identity or context is called ________.
Answer: Row Permissions

Row Permissions, part of DB2's Row and Column Access Control (RCAC), provide fine-grained security by filtering rows based on user identity or session context. This allows DB2 to restrict access to specific rows without requiring application-level logic, enhancing security for sensitive data.

28. A column defined to automatically generate sequential numeric values upon insertion is an ________ column.
Answer: Identity

An Identity column in DB2 is defined to automatically generate sequential numeric values upon row insertion, typically used for surrogate keys. It simplifies key management by eliminating the need for application logic to generate unique identifiers, with options to control starting values and increments.

 Schema Evolution & Performance

29. Which DB2 utility option allows reorganization of a tablespace with minimal application outage?
Answer: REORG SHRLEVEL CHANGE

The REORG SHRLEVEL CHANGE option allows reorganization of a tablespace with minimal application outage by permitting read and write access during most of the reorganization process. This ensures high availability, as applications can continue to access the tablespace while it is being reorganized to optimize performance.

30. True or False: Using views can help provide logical data independence during schema evolution.
Answer: True

Views in DB2 provide logical data independence by presenting a consistent interface to applications, even if the underlying table structures change during schema evolution. By modifying the view definition, DBAs can shield applications from changes in table design, reducing the need for application code changes.

31. Setting the `LOCKSIZE` option to ________ generally provides the highest level of concurrency but potentially higher overhead.
Answer: ROW

Setting the `LOCKSIZE` option to ROW provides the highest level of concurrency in DB2, as it locks only individual rows during transactions, minimizing contention. However, this can introduce higher overhead due to the increased number of locks managed, compared to page or table-level locking.

32. Which DB2 command is used to analyze the access path chosen by the optimizer for an SQL statement?
Answer: EXPLAIN

The EXPLAIN command in DB2 is used to analyze the access path chosen by the optimizer for an SQL statement. It generates detailed information about the query execution plan, including table accesses, index usage, and join methods, helping DBAs optimize performance by identifying inefficient queries.

 Temporal Data & Best Practices

33. Which type of temporal table automatically tracks the period when a row was physically present in the database?
Answer: System Time Table

A System Time Table in DB2 automatically tracks the period when a row was physically present in the database using system-generated timestamps. It is used for auditing and historical data tracking, with changes automatically recorded in an associated history table when rows are updated or deleted.

34. True or False: Business time periods represent the application-defined effective date range for a row.
Answer: True

Business time periods in DB2 represent the application-defined effective date range for a row, typically used to track when data is valid from a business perspective. This is distinct from system time, which tracks physical row existence, and is implemented using BUSINESS_TIME columns in temporal tables.

35. Using the `COMMENT ON` statement helps populate the DB2 ________.
Answer: Catalog

The `COMMENT ON` statement in DB2 is used to add descriptive remarks to database objects, storing this metadata in the DB2 Catalog (specifically in columns like REMARKS). This enhances documentation and helps DBAs and developers understand the purpose and structure of database objects.

36. Which Quality Assurance technique involves validating data integrity rules and relationships using DB2 utilities?
Answer: Database Verification

Database Verification is a Quality Assurance technique that uses DB2 utilities like CHECK DATA to validate data integrity rules and relationships. It ensures that constraints, such as referential integrity and check constraints, are enforced, identifying and reporting violations to maintain data quality.

37. Which routine maintenance activity is crucial for providing the DB2 optimizer with up-to-date information about data characteristics?
Answer: RUNSTATS

The RUNSTATS utility in DB2 is a critical maintenance activity that collects statistics about data distribution, cardinality, and other characteristics. These statistics are stored in the DB2 Catalog and used by the optimizer to choose efficient query execution plans, ensuring optimal performance.
