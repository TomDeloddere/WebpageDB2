Exercise 1: Create JCL for Dataset Allocation

Write JCL to allocate a new sequential dataset for storing DB2 utility output.

Requirements:

  • Allocate dataset MYUSER.DB2.OUTPUT
  • Use SYSDA unit, fixed-block format (FB), 80-byte records
  • Allocate 10 primary, 5 secondary cylinders, release unused space
  • Provide JCL
  • Explain the purpose of IEFBR14
//ALLOCDS  JOB (ACCT),'ALLOCATE DS',CLASS=A,MSGCLASS=X,NOTIFY=&SYSUID
//STEP1    EXEC PGM=IEFBR14
//OUTPUTDS DD DSN=MYUSER.DB2.OUTPUT,
//            DISP=(NEW,CATLG,DELETE),
//            UNIT=SYSDA,
//            SPACE=(CYL,(10,5),RLSE),
//            DCB=(RECFM=FB,LRECL=80,BLKSIZE=27920)

Explanation: The JCL uses IEFBR14, a null program, to trigger dataset allocation via the DD statement. The OUTPUTDS DD defines a new sequential dataset with specified attributes. IEFBR14 is used because it processes DD statements without executing a program, ideal for allocation/deletion tasks. The dataset can store DB2 utility output, with space released to optimize storage.

Exercise 2: JCL for RUNSTATS Utility

Create JCL to run the RUNSTATS utility on a tablespace (MYDB.MYTS).

Requirements:

  • Use DSNUTILB program, DB2 subsystem DB2X
  • Include STEPLIB, SYSPRINT, SYSUT1, and SYSIN
  • Update all statistics, allow concurrent access
  • Provide JCL
  • Explain why RUNSTATS is important
//RUNSTAT  JOB (ACCT),'RUNSTATS JOB',CLASS=B,MSGCLASS=X,NOTIFY=&SYSUID
//STEP010  EXEC PGM=DSNUTILB,REGION=0M,PARM='DB2X,RUNSTAT1'
//STEPLIB  DD DISP=SHR,DSN=DB2.V12.SDSNLOAD
//SYSPRINT DD SYSOUT=*
//SYSUT1   DD UNIT=SYSDA,SPACE=(CYL,(5,5))
//SYSIN    DD *
  RUNSTATS TABLESPACE MYDB.MYTS
    TABLE(ALL) INDEX(ALL)
    UPDATE ALL HISTORY ALL
    REPORT YES
    SHRLEVEL CHANGE
/*

Explanation: The JCL executes DSNUTILB to run RUNSTATS on MYDB.MYTS, updating table and index statistics. STEPLIB points to DB2 libraries, SYSPRINT captures output, SYSUT1 provides work space, and SYSIN specifies utility controls. RUNSTATS is critical for providing the DB2 optimizer with current statistics, ensuring efficient query execution plans and optimal performance.

Exercise 4: REXX Script to Query Table Count

Write a REXX script to count tables in a schema using DSNREXX.

Requirements:

  • Connect to DB2 subsystem DB2X
  • Query SYSIBM.SYSTABLES for schema MYSCHEMA
  • Handle SQL errors
  • Provide REXX script
  • Explain DSNREXX usage
/* REXX: Count tables in MYSCHEMA */
ADDRESS TSO "SUBCOM DSNREXX"
IF RC <> 0 THEN DO
   SAY "SUBCOM DSNREXX FAILED, RC="RC
   EXIT 8
END

ADDRESS DSNREXX "CONNECT DB2X"
IF SQLCODE <> 0 THEN DO
   SAY "CONNECT FAILED, SQLCODE="SQLCODE
   EXIT 12
END

SQL_STATEMENT = "SELECT COUNT(*) FROM SYSIBM.SYSTABLES WHERE CREATOR = ?"
HOST_VAR_CREATOR = 'MYSCHEMA'

ADDRESS DSNREXX "PREPARE S1 FROM :SQL_STATEMENT"
IF SQLCODE <> 0 THEN DO
   SAY "PREPARE FAILED, SQLCODE="SQLCODE
   EXIT 16
END

ADDRESS DSNREXX "EXECUTE S1 USING :HOST_VAR_CREATOR INTO :TABLE_COUNT"
IF SQLCODE = 0 THEN
   SAY "Number of tables in" HOST_VAR_CREATOR":" TABLE_COUNT
ELSE
   SAY "EXECUTE FAILED, SQLCODE="SQLCODE

ADDRESS DSNREXX "DISCONNECT"
EXIT 0

Explanation: The REXX script uses DSNREXX to connect to DB2X, execute a parameterized query, and retrieve the table count for MYSCHEMA. Error handling checks RC and SQLCODE. DSNREXX enables REXX to interact with DB2, supporting dynamic SQL, cursor operations, and command execution, ideal for automation tasks like catalog queries or generating utility JCL.

Exercise 5: Automate REORG with Conditional Execution

Create JCL to run REORG on a tablespace (MYDB.MYTS) with a conditional RUNSTATS step.

Requirements:

  • Use DSNUTILB for REORG and RUNSTATS
  • Run RUNSTATS only if REORG succeeds (RC=0)
  • Provide JCL
  • Explain conditional processing benefits
//REORGTS  JOB (ACCT),'REORG JOB',CLASS=A,MSGCLASS=X,NOTIFY=&SYSUID
//STEP1    EXEC PGM=DSNUTILB,PARM='DB2X,REORG1'
//STEPLIB  DD DISP=SHR,DSN=DB2.V12.SDSNLOAD
//SYSPRINT DD SYSOUT=*
//SYSUT1   DD UNIT=SYSDA,SPACE=(CYL,(10,5))
//SYSIN    DD *
  REORG TABLESPACE MYDB.MYTS
    LOG NO
/*
//STEP2    EXEC PGM=DSNUTILB,PARM='DB2X,STAT1',COND=(0,EQ,STEP1)
//SYSPRINT DD SYSOUT=*
//SYSUT1   DD UNIT=SYSDA,SPACE=(CYL,(5,5))
//SYSIN    DD *
  RUNSTATS TABLESPACE MYDB.MYTS
    TABLE(ALL) INDEX(ALL)
    UPDATE ALL HISTORY ALL
    SHRLEVEL CHANGE
/*

Explanation: The JCL runs REORG on MYDB.MYTS, followed by RUNSTATS only if REORG returns RC=0. The COND parameter ensures RUNSTATS executes only on success, preventing unnecessary statistics collection if REORG fails. Conditional processing enhances automation reliability, optimizes resource usage, and ensures dependent tasks execute in the correct sequence.

Exercise 6: Define a Job Scheduler Dependency

Define a job stream in a scheduler (e.g., IBM Workload Scheduler) to run COPY after LOAD completes.

Requirements:

  • Specify jobs LOADJOB and COPYJOB
  • Ensure COPYJOB runs only after LOADJOB succeeds
  • Provide a conceptual scheduler definition
  • Explain job scheduling benefits
SCHEDULE DB2_MAINT
  JOB LOADJOB
    JCL DSN=MYUSER.JCL.LOADJOB
    CLASS=A
    RUN DAILY AT 0200
  JOB COPYJOB
    JCL DSN=MYUSER.JCL.COPYJOB
    CLASS=A
    DEPENDS ON LOADJOB SUCCESS
    RUN DAILY
END

Explanation: The scheduler definition ensures COPYJOB runs only after LOADJOB completes successfully, enforcing dependency. Job schedulers like IBM Workload Scheduler manage complex job streams, handle dependencies, optimize batch windows, and provide monitoring, reducing manual coordination and ensuring reliable execution of DB2 maintenance tasks.

Exercise 7: Configure System Automation for DB2 Messages

Create a System Automation (SA z/OS) policy to react to a DB2 tablespace space warning message.

Requirements:

  • Monitor for DSNT501I message (space issue)
  • Trigger a notification and start an extension job
  • Provide conceptual policy
  • Explain monitoring importance
DEFINE POLICY DB2_SPACE_MONITOR
  WHEN MESSAGE ID = 'DSNT501I'
    EXECUTE COMMAND 'MVS S EXTEND_SPACE_JOB'
    NOTIFY OPERATOR 'DB2 Tablespace space warning detected, extension job started.'
END

Explanation: The SA z/OS policy monitors for DSNT501I, triggering a job to extend space and notifying operators. Monitoring console messages enables proactive issue detection, ensuring timely responses to critical conditions like space shortages, maintaining DB2 availability, and reducing downtime risks.

Exercise 8: Documented and Modular JCL

Create a documented JCL procedure for CHECK DATA with modular error handling.

Requirements:

  • Include comments and error notification step
  • Use a PROC for reusability
  • Provide JCL
  • Explain documentation benefits
//CHECKPRC PROC TSNAME='MYDB.MYTS',SSID='DB2X'
//* Purpose: Run CHECK DATA on specified tablespace
//* Parameters: TSNAME - Tablespace name, SSID - DB2 subsystem
//CHECK    EXEC PGM=DSNUTILB,PARM='&SSID.,CHECK1'
//STEPLIB  DD DISP=SHR,DSN=DB2.V12.SDSNLOAD
//SYSPRINT DD SYSOUT=*
//SYSUT1   DD UNIT=SYSDA,SPACE=(CYL,(5,5))
//SYSIN    DD *
  CHECK DATA TABLESPACE &TSNAME.
    SCOPE ALL
/*
//* Error notification if CHECK fails
//ERRNOTIF EXEC PGM=IKJEFT01,COND=(4,GT,CHECK)
//SYSTSPRT DD SYSOUT=*
//SYSTSIN  DD *
  NOTIFY 'CHECK DATA FAILED FOR &TSNAME., RC=&RC'
/*
//         PEND
//*
//CHECKJOB JOB (ACCT),'CHECK DATA',CLASS=A,MSGCLASS=X,NOTIFY=&SYSUID
//RUNCHECK EXEC PROC=CHECKPRC,TSNAME='PROD.CUST',SSID='DB2A'

Explanation: The PROC modularizes CHECK DATA execution, with comments explaining purpose and parameters. An error notification step runs if CHECK fails (RC>4). Documentation clarifies usage, aids maintenance, and supports team collaboration, reducing errors during updates or debugging.

Exercise 9: Use TEMPLATE and LISTDEF for COPY

Create JCL with TEMPLATE and LISTDEF to copy multiple tablespaces.

Requirements:

  • Copy all tablespaces in database MYDB
  • Use dynamic dataset naming with date
  • Provide JCL
  • Explain TEMPLATE and LISTDEF advantages
//COPYTS   JOB (ACCT),'COPY JOB',CLASS=A,MSGCLASS=X,NOTIFY=&SYSUID
//STEP1    EXEC PGM=DSNUTILB,PARM='DB2X,COPY1'
//STEPLIB  DD DISP=SHR,DSN=DB2.V12.SDSNLOAD
//SYSPRINT DD SYSOUT=*
//SYSIN    DD *
  LISTDEF MYDB_TBS INCLUDE TABLESPACES DATABASE MYDB

  TEMPLATE COPYDS
    DSN 'BACKUP.&DB..&TS..D&JDATE.'
    UNIT SYSDA SPACE=(CYL,(10,5)) DISP=(NEW,CATLG)

  COPY LIST MYDB_TBS
    COPYDDN(COPYDS)
    FULL YES
    SHRLEVEL CHANGE
/*

Explanation: LISTDEF defines a list of all tablespaces in MYDB, and TEMPLATE creates dynamic dataset names incorporating the date. The COPY utility processes all listed tablespaces. These features simplify JCL for large object sets, ensure consistent naming, and reduce manual configuration, enhancing automation scalability.
